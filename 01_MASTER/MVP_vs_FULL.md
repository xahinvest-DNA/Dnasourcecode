# MVP vs Full

| Area | MVP | Later Expansion |
| --- | --- | --- |
| Domain | Money/income only | Other domains such as self-worth, visibility, relationships |
| Flow | One completed restructuring cycle | Multi-cycle programs and longer tracks |
| Actioning | One micro-action per cycle | Action sequences, streaks, and progressive plans |
| Diagnosis | One leading mechanism hypothesis | Multiple competing hypotheses and confidence scoring |
| Memory | Basic cycle history | Deeper pattern mining and cross-cycle analytics |
| UX | Linear mini-app flow | Adaptive branching and richer personalization |
| Output | Structured reframing and check-in | Dashboards, summaries, and manager/operator tooling |

## MVP Rule
- Prefer narrow depth over broad surface area.
- Do not open later-expansion items until one working cycle is stable.
