# Check-In Engine

## Responsibility
Capture post-action evidence and determine whether the cycle closed, partially shifted, or remained unchanged.

## Inputs
- Action plan
- User completion response
- Observed result
- Reported internal reaction

## Outputs
- Completion status
- Shift markers
- Old-cycle reappearance note
- Cycle close recommendation

## Non-Goals
- Long reflective journaling
- New diagnosis without reference to the assigned action
- Automatic opening of the next cycle without manager-approved packet logic

## Boundary Rule
- Check-in must ask for evidence of action and observed effect, not just subjective liking.
