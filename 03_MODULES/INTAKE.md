# Intake Module

## Responsibility
Capture the user's live money/income problem in natural language and convert it into a structured intake record.

## Inputs
- Free-text description of the current problem
- Optional examples of repeated situations
- Optional statement of desired change

## Outputs
- Problem summary
- Repeated pattern summary
- Candidate belief language
- Candidate behavior clues
- Intake completeness flag

## Non-Goals
- Final diagnosis
- Advice generation
- Multi-session planning

## Boundary Rule
- Intake must collect enough signal for diagnosis without turning into a long therapy interview.
