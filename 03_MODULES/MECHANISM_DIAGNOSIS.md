# Mechanism Diagnosis Module

## Purpose
Produce one operational `Diagnosis Output` from the `Intake Record` so the product can show one leading hidden poverty mechanism and move into old-cycle presentation.

## Upstream Input
- `Intake Record`
- Communication DNA support signals
- Prior cycle history when available

## Owned Transformation
- Select one leading hidden poverty mechanism hypothesis
- Translate intake clues into one old-belief statement
- Surface hidden prohibition or safety issue when materially relevant
- Prepare the minimum artifact needed for old-cycle presentation

## Validation
- Confirm the `Intake Record` is complete enough for diagnosis
- Confirm the selected hypothesis is singular, not a list
- Confirm the output does not include restructuring or action content

## Interpretation
- Primary interpretation happens here
- This module interprets intake clues and DNA support into one leading mechanism hypothesis
- This module may use prior history as context, but it must still return one current-cycle diagnosis

## Output Artifact
`Diagnosis Output`

### Required Fields
- leading mechanism hypothesis
- old belief statement
- attention bias clue
- behavior pattern clue
- reinforcement logic

### Optional Field
- hidden prohibition statement

## Exit Condition
- One diagnosis artifact exists and is specific enough to drive old-cycle mapping

## Must Not Do
- Construct the new cycle
- Generate a new belief
- Assign an action
- Return multiple competing diagnoses in MVP
- Behave as final psychological truth

## Boundary Rule
- Diagnosis owns hidden-mechanism interpretation.
- Diagnosis does not restructure or assign action.
