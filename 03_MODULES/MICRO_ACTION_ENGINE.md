# Micro Action Engine

## Responsibility
Generate exactly one bounded action that tests the new cycle in the real world.

## Inputs
- New cycle output
- User constraints and context
- Prior completion history

## Outputs
- One concrete action
- Completion criteria
- Timeframe
- Failure-risk note

## Non-Goals
- Action lists
- Daily habit programs
- Broad plans with several moving parts

## Boundary Rule
- If the action cannot be completed as a single bounded step, it is too large for the cycle.
