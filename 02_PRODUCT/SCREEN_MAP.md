# Screen Map

| Screen | Purpose | Key Output |
| --- | --- | --- |
| `S-01 Entry` | Frame the app as a trainer for hidden mechanism restructuring | User starts intake |
| `S-02 Intake` | Capture the user's live money problem in natural language | Structured intake record |
| `S-03 Diagnosis` | Present the leading hidden mechanism hypothesis | Mechanism hypothesis |
| `S-04 Old Cycle` | Show the current self-reinforcing cycle | Old cycle map |
| `S-05 New Cycle` | Present admissible reframing and target cycle | New belief and new cycle map |
| `S-06 One Action` | Assign exactly one bounded micro-action | Action plan |
| `S-07 Check-In` | Gather what happened after the action | Check-in record |
| `S-08 Progress` | Show completed cycles and visible shifts | Progress snapshot |

## Screen Ownership Rule
- One primary output per screen.
- If a screen cannot be tied to one primary output, split or remove it.
