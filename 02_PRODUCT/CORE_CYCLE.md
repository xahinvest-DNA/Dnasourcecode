# Core Cycle

## Product Thesis
The product intervenes in one repeating money-related cycle:
`belief -> attention -> behavior -> result -> reinforcement`

The MVP does not try to change the whole person.
It tries to complete one restructuring cycle that makes this loop visible, testable, and partially replaceable.

## Old Cycle Definition

| Link | Definition | MVP Product Task |
| --- | --- | --- |
| Belief | What the user currently treats as true about money/income | Name it in recognizable language |
| Attention | What the user notices and ignores because of that belief | Make the bias visible |
| Behavior | What the user repeatedly does or avoids doing | Show the pattern created by the belief |
| Result | What outcome this behavior tends to create | Show the repeated money/income consequence |
| Reinforcement | How the result appears to prove the old belief again | Close the loop explicitly |

## New Cycle Definition
The replacement cycle in MVP is:
`new belief -> new attention target -> new behavior direction -> new result marker -> new reinforcement`

This is not an extreme opposite belief.
It is one admissible restructuring that the user can test through one bounded action.

## Canonical Example

### Old Cycle
- Belief: more money is possible only through strain
- Attention: notices workload and obstacles, ignores leverage and pricing
- Behavior: overworks, delays asking for more, stays underpriced
- Result: low or unstable income
- Reinforcement: sees the result as proof that money is heavy and scarce

### New Cycle
- New belief: income can grow through clearer value and cleaner asks, not only through more strain
- New attention target: notice places where value is already created but not claimed
- New behavior direction: make one clearer ask instead of adding more effort
- Desired result marker: one moment of cleaner claiming or pricing
- New reinforcement: a concrete shift can exist without extreme overload

## Value Unit
- One completed restructuring cycle is the unit of value.
- A cycle is completed only after check-in and progress fixation.
