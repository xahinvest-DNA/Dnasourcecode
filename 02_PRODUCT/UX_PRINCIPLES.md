# UX Principles

- Keep the user inside one concrete cycle, not a broad self-development space.
- Use operational language over inspirational language.
- Show structure before advice.
- Present one leading hypothesis, not a list of interpretations.
- Never assign more than one real-world action in a cycle.
- Ask for check-in evidence, not mood-only feedback.
- Preserve psychological admissibility: reframing must be believable enough to act on.
- Prefer short, high-signal screens over dense reflective exercises.
