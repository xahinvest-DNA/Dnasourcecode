# Implementation Rules

- Codex executes bounded packets only.
- Codex does not redefine project direction independently.
- Use the manager-led model at all times.
- Keep exactly one active packet in `05_CODEX/NEXT_TASK.md`.
- Synchronize state files after every substantive packet.
- Avoid scope explosion and broad speculative implementation.
- Do not open runtime or product implementation while the active packet is documentation-only.
- Prefer small, reviewable changes over sweeping redesign.
- Treat the repository as the source-of-truth system.
